1. なぜ統合が必要か

解析モデルでは、実際には1本の部材（母屋・胴縁・弦材など）が、小梁の取り付きや床の分割に 合わせて複数の要素に分割されていることがよくあります。 解析上はそれで問題ありませんが、断面算定で座屈長さを取るときは「実際の1本」の長さが必要です。 分割要素の長さをそのまま座屈長さにすると、実際より短い＝危険側の評価になってしまいます。

2. 前提となる用語

用語意味
節点（ノード）モデル上の点。座標 (X, Y, Z) を持つ。要素は必ず2つの節点を結ぶ
要素（エレメント）解析の最小単位の線材。I 端（始点）と J 端（終点）を持ち、タイプ（BEAM＝梁、TENSTR＝引張専用トラス等）・材料・断面・β角（断面の回転角）が割り当てられている
部材（メンバー）このツールが要素をまとめて作る単位。設計者が「1本」と認識する材に対応させることを狙う
梁端解放（ピン）要素の端部で曲げ等を伝えない設定（MGT の *FRAME-RLS）。フランジ継手のないボルトピン接合などに使われる
支点変位や回転を拘束された節点（MGT の *CONSTRAINT）

3. 判定の全体像

判定は「節点ごとに、そこで出会う2要素がつながるかどうか」を見る、というシンプルな考え方です。

  1. 全要素について、両端の節点を調べ、「節点 → その節点に取り付く要素」の対応表を作る
  2. 各節点で、後述の5条件をすべて満たす要素ペアを「連結」と判定する
  3. 連結judgの結果をグループ化（Union-Find、6章）し、つながった要素の集まり＝1部材とする
  4. どの要素とも連結しなかった要素は、それ単独で1部材になる

ポイントは、部材の形を先に仮定しないことです。節点単位の局所的な判定を積み上げるだけで、 何要素に分割されていても（10分割でも）自然に1本にまとまります。

4. 節点での連結条件（5つ）

ある節点で出会う2つの要素は、次の5条件をすべて満たすときだけ連結されます。

条件 1同じ属性であること

要素タイプ（BEAM / TENSTR 等）・材料・断面・β角（断面の回転）がすべて一致すること。 断面が H-300 から H-250 に変わる点は継手＝部材の切れ目とみなします。

条件 2同一直線上にあること

2要素の方向ベクトルのなす角が約 0.8°以内（cos > 0.9999）であること。 折れ曲がる点（山形の頂点など）では切れます。座標誤差程度の微小な角度は許容されます。

比較の際は「節点を通過する向き」に揃えてから角度を測ります。要素の I→J の向きは モデリング順で決まっていてバラバラなので、節点に J 端で到達する要素はそのまま、 I 端で到達する要素は向きを反転して比較します（これにより I-I、I-J、J-I、J-J の どの組み合わせでも正しく判定できます）。

条件 3同属性の要素がその節点にちょうど2本であること

同じ属性の要素が節点に3本以上集まっている場合、どれとどれが「1本」なのか 幾何だけでは決められない（X形の交差など）ため、安全側としてその節点では統合しません。

条件 4節点が支点でないこと

支点（*CONSTRAINT の節点）は部材が横方向に押さえられる点なので、 座屈長さの区切りとしてそこで必ず切ります。GUI のオプションで OFF にもできます。

条件 5共有節点側の要素端にピン（梁端解放）がないこと

2要素のどちらかが、その共有節点側の端部に梁端解放（*FRAME-RLS のいずれかの成分の解放）を 持っていれば、そこは構造的な継ぎ目＝部材の切れ目とみなします。 判定するのは共有節点側の端部だけで、部材の外側の端部にピンがあっても統合は妨げません。 GUI のオプションで OFF にもできます。

注意（混同しやすい点）: 見るのは通し材（統合しようとしている2要素）自身の端部条件だけです。 その節点に取り付く小梁やブレースの側の端部がピン接合でも、通し材の統合には影響しません。 逆に、通し材自身の要素端にピンが設定されていれば、そこは継手＝1本の終わりとして必ず切ります。

5. ケース別の判定例

ケース判定理由
直線上に同断面が連続し、中間節点に小梁やブレースが取り付く統合する取り付く相手は属性が違うので条件1で対象外。本体2本は5条件を満たす（条件3の「同属性2本」も満たす）
要素のモデリング向き（I→J）が互い違い統合する条件2で向きを節点通過方向に揃えてから比較するため影響しない
中間節点が支点（柱頭・壁上など）切る条件4。座屈長さは支点間で取る
中間節点側の端部にピン（梁端解放）切る条件5。継手位置＝1本の終わり
断面や材料が変わる（H-300 → H-250）切る条件1。継手とみなす
山形の頂点で折れる（登り梁どうし）切る条件2。直線でない
同断面ブレースがX形に交差する節点切る条件3。同属性が4本集まり、ペアが曖昧

6. グループ化の仕組み（Union-Find）

節点ごとの「この2要素は連結」という判定結果を部材にまとめる処理には、 Union-Find（素集合データ構造）という定番のアルゴリズムを使っています。

  1. 最初、全要素は「自分ひとりのグループ」に属する
  2. 「要素AとBは連結」と判定されるたびに、Aのグループ と Bのグループ を合併（union）する
  3. すべての節点を処理し終えたら、同じグループに属する要素の集まりが1部材

たとえば図1の要素 101–102 が節点で連結、102–103 が別の節点で連結なら、 101・102・103 は自動的に同じグループ＝1部材になります。連結判定は常に隣どうしの 局所判定だけで済み、順番にも依存しません。

7. 座屈長さと断面力の集計

部材長さ（＝座屈長さ）

部材の長さは構成要素の長さの合計です。統合条件に直線性が入っているため、 これは両端節点間の直線距離と一致します（ツール内部では両者の一致を検証済み）。 出力される「部材長さ L」を、強軸・弱軸の座屈長さ（lk）としてそのまま、 あるいは設計判断で係数を掛けて使ってください。

断面力の max/min

部材一覧の断面力は、荷重ケース（LC）ごとに、部材を構成する全要素の全出力位置 （I 端・中間点 1/4・CNT・3/4・J 端）から N・Qy・Qz・Mt・My・Mz それぞれの最大値と最小値を 拾ったものです。中間の分割節点の値も漏れなく包絡されます。

符号について: 統合部材の中にモデリング向き（I→J）が逆の要素が混ざっている場合、 せん断や曲げの符号が要素間で反転して混ざります。max/min の絶対値の包絡としては安全側に 保存されますが、符号の向きを厳密に使うときは「要素データ」シートで要素単位に確認してください。

8. 限界と注意点

事項内容
3本以上の同属性が集まる節点統合しない（安全側）。通し材が実在する場合は、出力の「構成要素ID」を見て手動で長さを合算する
曲線状の材（折れ線アーチ等）折れ点ごとに切れる。アーチ弦材の座屈長さは別途設計判断が必要
ピンや支点の設定漏れ判定はモデルの入力（FRAME-RLS / CONSTRAINT）に従う。モデル側の設定が実情と違えば結果も違う
横補剛材による座屈長さの低減中間に取り付く小梁・母屋を横補剛とみなすかは設計判断のため、ツールは自動では切らない（=通し長さで安全側）。補剛点で切りたい場合はモデルで支点やピンを設定するか、Excel 上で長さを調整する
座屈長さの強軸/弱軸の区別出力は幾何的な通し長さ1本。lky / lkz の使い分けは設計者が行う
